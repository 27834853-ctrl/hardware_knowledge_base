# 贡献者公约行为准则 Contributor Covenant Code of Conduct

## 我们的承诺

为了促进一个开放和友好的环境，我们作为贡献者和维护者承诺：无论年龄、体型、身体健全与否、民族、性别认同和表达（方式）、经验水平、教育程度、社会地位、国籍、个人外貌、种族、宗教或性取向如何，参与我们项目和社区的每个人都免遭骚扰。

## 我们的标准

有助于创造积极环境的行为包括：

* 使用友好和包容性语言
* 尊重不同的观点和经验
* 优雅地接受建设性批评
* 关注什么对社区最有利
* 对其他社区成员表示同理心
* 专注于技术和知识的分享

不可接受的行为包括：

* 使用性化的语言或图像，以及不受欢迎的性关注或挑逗
* 捣乱、侮辱性或贬损性的评论，以及人身攻击或政治攻击
* 公开或私下的骚扰
* 未经明确许可，发布他人的私人信息，如物理地址或电子地址
* 其他在专业场合中可以被合理认为不适当的行为

## 我们的责任

项目维护者有责任澄清可接受行为的标准，并针对任何不可接受的行为采取适当和公平的纠正措施。

项目维护者有权利和责任删除、编辑或拒绝与本行为准则不符的评论、提交、代码、wiki编辑、问题和其他贡献，或暂时或永久禁止任何他们认为有不适当、威胁、冒犯或有害行为的贡献者。

## 适用范围

本行为准则适用于项目空间和公共空间，当个人代表项目或其社区时。代表项目或社区的示例包括使用官方项目电子邮件地址、通过官方社交媒体帐户发布信息，或在线上或线下活动中担任指定代表。项目的表示可以由项目维护者进一步定义和澄清。

## 执行

可以通过GitHub Issues联系项目团队来报告辱骂、骚扰或其他不可接受的行为。所有投诉都将被审查和调查，并将导致必要的和适当的回应。项目团队有义务对事件报告者保密。具体执行政策的更多细节可能会单独发布。

不真诚地遵守或执行行为准则的项目维护者可能会面临项目领导层其他成员确定的临时或永久性后果。

## 技术社区特别说明

### 专业技术讨论

在技术讨论中，我们鼓励：

✅ **建设性批评**: 基于事实和标准的技术讨论
✅ **知识分享**: 分享经验、案例和最佳实践
✅ **相互学习**: 尊重不同经验水平的贡献者
✅ **文明辩论**: 就技术方案进行专业辩论
✅ **引用来源**: 引用权威标准和文档支持观点

❌ **避免以下行为**:
- 人身攻击或贬低他人的技术水平
- 不尊重初学者的问题
- 无建设性的批评
- 傲慢或轻蔑的态度
- 传播未经验证的技术信息

### 问题报告和讨论

- 保持耐心和友好
- 提供详细的技术信息
- 尊重他人的时间
- 避免重复已解决的问题
- 感谢帮助您的人

### 代码审查

- 关注代码，而不是编写者
- 清楚解释建议的原因
- 寻求理解而非批评
- 接受不同的编程风格
- 认可好的代码
